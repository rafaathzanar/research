# Module 2: Marketing Automation and Campaign Management

## What this is

A comparative evaluation of fixed, trigger-based, and hybrid campaign
automation strategies under simulated cold-start marketing conditions. An
event-driven simulator, an ML routing model, and a strategy-comparison
pipeline are used to measure engagement, conversion efficiency, and
operational complexity trade-offs. Part of a four-module group project on
AI-powered digital marketing orchestration.

## Research question

"Which automation strategy gives the best balance between engagement,
conversion efficiency, and operational complexity?"

## Prerequisites

- Python 3.10+
- Virtual environment (`.venv/`) — bootstrapped via pip
- Dataset at `data/raw/digital_marketing_conversion.csv` (Kaggle synthetic
  digital marketing dataset; 8000 users)
- Segmentation input at `data/processed/user_segments.csv` (provided by
  Module 1 — NOT generated by this module)

## Setup

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## Reproducing all results

```bash
python main.py            # runs entire pipeline
python main.py --list     # shows available phases
python main.py --phase ml # runs one phase only
```

## Key findings

Trigger dominates per-send conversion efficiency across every tested
condition; hybrid dominates user-level conversion reach; no single strategy
dominates both metrics simultaneously. Fixed is the complexity floor
(`OPERATIONAL_COMPLEXITY=7`) but the efficiency floor too. These rankings
were validated across 20 random seeds, 5 signal-sparsity levels (plus 4
extreme levels up to 0.99), and 4 `INTENT_UPLIFT` values (1.5–3.0) — the
ordering never changed in any of these robustness checks.
